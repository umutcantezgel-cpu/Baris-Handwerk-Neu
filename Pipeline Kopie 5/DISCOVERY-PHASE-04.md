ULTRATHINK-LANGZEITTASK 4.1: TYPE-SYSTEM VOLLSTAENDIGE ERFASSUNG
─────────────────────────────────────────────────────────────────────────────
Scanne ALLE .ts/.tsx Dateien nach Type-Definitionen:
  Interfaces: Name, Properties (Typ, Optional/Required), Extends.
  Type Aliases: Name, Definition, Generics.
  Enums: Name, Werte, Verwendung.
  Zod Schemas: Name, Shape, Validierungsregeln.
Fuer JEDE Type-Definition:
  Wo wird sie definiert? (Zentral in /types oder verstreut?)
  Wo wird sie verwendet? (Welche Komponenten, Hooks, Utils?)
  Ist sie exportiert oder nur lokal?
  Gibt es Duplikate? (Gleicher Name in verschiedenen Dateien?)
  Gibt es Konflikte? (Gleiches Interface, unterschiedliche Properties?)

ULTRATHINK-LANGZEITTASK 4.2: DATENQUELLEN-KARTIERUNG
─────────────────────────────────────────────────────────────────────────────
Identifiziere ALLE Datenquellen:
  Statische Daten: /lib/data/*.ts, /constants/*.ts, JSON-Dateien.
  API-Aufrufe: fetch(), axios, React Query, SWR, tRPC.
  CMS-Anbindung: Contentful, Sanity, Strapi, Prismic? (Falls vorhanden trotz Verbot.)
  Datenbank: Prisma, Drizzle, direkte SQL?
  Externe Services: Formspree, Calendly, Analytics.
Fuer JEDE Datenquelle:
  Format der Daten (TypeScript-Objekte, JSON, API-Response).
  Typisierung (strikt typisiert oder any/unknown?).
  Validierung (Zod, io-ts, manuell, gar nicht?).
  Fehlerbehandlung bei Daten-Abruf.

ULTRATHINK-LANGZEITTASK 4.3: DATENFLUSS-ANALYSE (DATA FLOW MAPPING)
─────────────────────────────────────────────────────────────────────────────
Trace den Datenfluss fuer JEDEN Datentyp:
  QUELLE → TRANSFORMATION → KOMPONENTE → RENDERING.
  Beispiel: Service-Daten aus /lib/data/services.ts →
    getServiceBySlug() in /lib/utils.ts →
    ServicePage in /app/services/[slug]/page.tsx →
    ServiceCard Komponente rendert title, description, image.
Identifiziere:
  LUECKEN: Wo werden Daten geladen aber nie verwendet?
  TRANSFORMATIONS-KETTEN: Wo werden Daten mehrfach transformiert?
  TYP-BRUECKE: Wo geht Typ-Information verloren? (any-Casts, JSON.parse)
  NULLABLE KETTEN: Wo kann ein null/undefined unkontrolliert durchfliessen?

ULTRATHINK-LANGZEITTASK 4.4: REFERENTIELLE INTEGRITAET
─────────────────────────────────────────────────────────────────────────────
Erstelle die vollstaendige Referenz-Matrix:
  Wenn Entity A auf Entity B verweist (via Slug, ID, Key):
    Existiert B? Ist die Referenz bidirektional? Kann B geloescht werden?
  Pruefe auf:
    Verwaiste Referenzen (A zeigt auf nicht-existierendes B).
    Doppelte Slugs/IDs (Nicht-Eindeutigkeit).
    Typ-Mismatches in Referenzen (string vs. number).
    Fehlende Null-Checks bei optionalen Referenzen.
